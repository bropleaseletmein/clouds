Дано: 
- Данные лабораторной работы 1.

## Ход работы

Воспользуемся моделью, которую использовали для выполнения ЛР1. 

так же для откроем сайт с документацией `https://learn.microsoft.com/en-us/azure/?product=popular`

и так же для поиска информации смотрим в формате "Meter Category" azure documentation

тк мы сохраняем моделью ЛР1, то у нас будут такие же `IT Tower`,	`Service Family`:

IT Tower:
- Cloud Services
- Database
- Compute
- Storage
- Networking
  
Service Family:
- Analytics
- Compute
- Database
- Application Services
- Networking
- Storage&Content Delivery
- Internet of Things (IoT)
- Management Tools
- Machine Learning

определяем Service Type по документации

Service Sub Type берем (если есть) из ЛР1, смотрим что подходит. если такого нет - ставим новый. в моем варианте не все было, например из ЛР1 использовалось `GPU-powered acceleration`, `Backup` и некоторые другие.

в итоге путем сравненимя и сопоставленния, а так же используя общую модель получаем табличку. так же в итоге у нас есть единая кросс-провайдерная сервисна модель.
