## Начало

для начала ознакомимся с терминами IaaS, PaaS, SaaS:

- IaaS (Infrastructure as a Service) - инфраструктура как услуга. Поставщик IaaS предоставляет нам инфраструктуру - физический сервер, интернет, оборудование - все это уже есть. При использовании мы сами настриваем остальное по необходимости - ОС, приложения и тп.
- PaaS (Platform as a Service) - облачная платформа как услуга. Поставщик PaaS. Отличие PaaS от IaaS в том, что уже есть определенные инструменты, например: система управления базами данных и тп. Их нужно настроить под потребности компании, но не надо строить с нуля. При нет доступа к     операционной системе, настройкам виртуальных серверов.
- SaaS (Software as a service) - программное обеспечение как сервис. SaaS — это полностью настроенная и готовая к работе программа, выполняющая определенные функции. Доступ к ней идет через интернет, а сама программа работает на мощностях виртуальных серверов, поэтому не нагружает личный компьютер.

## Ход работы

идем на сайт с документацией `https://docs.aws.amazon.com/`

ищем нужные сервисы по Product Code. если нет - гуглим в формате "Product Code" aws documentation. удалось найти все сервисы используя такой формат поиска

фиксируем

- IT Tower

- Service Family - группа сервсиов

- Service Type - имя сервиса

- Service Sub Type - что конкертно делает этот сервис

- Service Usage Type - что именно в сервсие используется

несколько примеров 

IT Tower:
- Cloud Services
- Database

Service Family:
- Application Services
- Developer Tools
- Internet of Things (IoT)

Service Type:
- Amazon DocumentDB
- AWS IoT Core

Service Sub Type:
- Broker
- CI/CD
- GPU-powered acceleration

Service Usage Type:
- stop instance
- batch prediction
